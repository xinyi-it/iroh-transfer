# iroh-transfer 传输核心设计文档

## 项目概述

基于iroh CLI的P2P文件传输桌面应用，使用Tauri + Web前端 + iroh CLI三层架构。

## 技术选型

- 框架：Tauri
- 前端：HTML/CSS/JS（后续可换Vue/React）
- 后端：Rust薄层，shell调iroh CLI
- 传输引擎：iroh CLI 0.28.1（内置打包）
- 目标平台：Linux / macOS / Windows（桌面三端）

## 架构

```
前端界面 → 传输接口（抽象层）→ iroh CLI（现在）
                              → iroh Rust SDK（中期）
                              → 其他引擎（未来移动端）
```

三层结构：
1. 前端：拖拽发文件、输入票据接收、进度显示、历史记录
2. Rust后端：薄层，执行iroh命令、解析输出、返回状态
3. iroh CLI：内置打包，用户不用单独装

## 功能模块

### 1. 节点管理
- 应用启动时自动启动iroh节点（iroh start）
- 显示当前节点ID和状态
- 应用关闭时自动停止节点

### 2. 发送文件
- 拖拽或选择文件
- 调用iroh blobs add上传
- 生成票据，一键复制或生成二维码
- 支持多文件、文件夹

### 3. 接收文件
- 粘贴票据或扫二维码
- 调用iroh blobs get下载
- 调用iroh blobs export导出到指定目录
- 支持选择保存路径

### 4. 传输管理
- 传输进度显示（从iroh命令行输出解析）
- 传输历史记录
- 传输状态：等待中/传输中/完成/失败

## 数据流

### 发送流程
用户拖入文件 → 前端传文件路径给Rust后端
→ Rust执行 iroh blobs add <文件路径>
→ 解析输出拿到Blob ID和票据
→ 返回给前端显示票据+生成二维码

### 接收流程
用户粘贴票据 → 前端传票据给Rust后端
→ Rust执行 iroh blobs get <票据>
→ 解析输出拿到下载进度
→ 实时推送给前端更新进度条
→ 下载完成后执行 iroh blobs export <BlobID> <保存路径>
→ 返回完成状态给前端

### 进度获取
iroh blobs get 输出中包含传输字节数/总字节数
→ Rust后端实时解析输出流
→ 通过Tauri事件系统推送到前端
→ 前端更新进度条

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| iroh节点启动失败 | 提示"节点启动失败，请检查网络"，提供重试按钮 |
| 票据无效/过期 | 提示"票据无效，请重新获取" |
| 传输中断 | 显示"传输中断"，提供断点续传按钮 |
| 磁盘空间不足 | 发送前检查文件大小，接收前检查剩余空间 |
| iroh CLI进程崩溃 | 自动重启节点，恢复传输状态 |
| 对端离线 | 提示"对端不在线，文件已缓存，对端上线后可继续接收" |
